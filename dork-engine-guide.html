WHAT IS IT?
DORK//ENGINE is an advanced Google dorking assistant for OSINT (Open Source Intelligence) and authorized security research. It generates pre-built search queries that find sensitive exposed data on target domains.
🛡️
WHO IS IT FOR?
Security researchers, penetration testers, bug bounty hunters, and OSINT analysts who need to enumerate publicly exposed information on domains they are authorized to test.
⚠️
IMPORTANT
This tool generates Google search queries only. It does not perform any hacking. Always ensure you have explicit written permission before conducting reconnaissance on any target domain.

SECTION 04
⛓ CHAIN BUILDER
⚡ WHAT IS A CHAIN?
A chain is a queue of dorks you want to run in sequence. Instead of clicking each dork individually, you add multiple dorks from different categories to the chain, then execute them all at once with an animated progress bar. Perfect for running a full recon scan against a target.
1
ADDING TO CHAIN
There are 4 ways to add dorks to the chain: hover a dork in Generator and click ⛓, click ⛓ CHAIN on any result card, use ⛓ CHAIN ALL in Search, or use the AI Generate tab's "⛓ CHAIN ALL" button.
2
EXECUTE CHAIN
Click ▶ EXECUTE CHAIN. Each dork runs with a 120ms delay between them, showing an animated progress bar. All results are added to the Results panel simultaneously.
3
MANAGE CHAIN
Each chain item has a ▶ button to run just that one dork, ⎘ to copy the query, and ✕ to remove it. Use ↓ EXPORT CHAIN to save the chain as a text file.

✏ CUSTOM BUILDER TAB

Create, save, and manage your own dork templates. Use {"{target}"} as a placeholder — it gets replaced with your target domain when you run the dork.

1
FILL IN THE FORM
Enter a label (name), choose a risk level, set a category name, write your template, and add comma-separated tags for filtering in Search.
2
LIVE PREVIEW
As you type your template, the PREVIEW box shows the resolved query with your target domain substituted in. This lets you verify the query looks correct before running it.
3
3 ACTIONS
▶ RUN NOW — immediately executes and adds to results. ⛓ ADD TO CHAIN — queues it. ♥ SAVE DORK — persists it in the session for reuse.
EXAMPLE CUSTOM DORK
Label: Password Exposure Hunt Risk: critical Category: Secrets Template: site:{target} intext:"password" OR intext:"passwd" filetype:txt OR filetype:log Tags: secrets, passwords, files → Resolves to: site:example.com intext:"password" OR intext:"passwd" filetype:txt OR filetype:log
IMPORT DORK
Paste a raw dork query string OR a JSON object into the IMPORT DORK field on the right side. Supports formats:

Raw: site:{"{target}"} intext:"api_key"

JSON: {"{"}"label":"...","template":"...","risk":"..."{"}"}
Great for importing dorks from external collections or teammates

Human-readable report with risk labels, queries, URLs, and metadata. Best for sharing with non-technical stakeholders.

Spreadsheet-compatible with columns: Label, Risk, Category, Query, URL, Target, Time. Import into Excel, Sheets, or databases.

Structured JSON array. Ideal for programmatic processing, integrating with SIEM tools, or building automation scripts.
📝

Markdown report with headers for each dork. Perfect for GitHub, Notion, Obsidian, or any documentation platform.

SECTION 09
◎ RADAR TAB

The Radar tab provides a visual threat map of your scan results. A spider/radar chart shows how many dorks you've run per category, weighted by risk severity.

1
LEFT PANEL — THREAT STATS
Shows a live breakdown: total queries run, plus a count for each risk level (Critical, High, Medium, Low). Below that, each category shows its result count and a proportional bar.
2
RADAR CHART
Each axis represents a category. The polygon's shape shows where your recon is concentrated. A larger polygon on the "Secrets" axis means more credential-related dorks were run. Numbers in parentheses show result count per category.
3
REFRESH RADAR
Click ↺ REFRESH RADAR to redraw the chart with the latest results. The radar is also redrawn automatically each time you switch to this tab.

SECTION 11
TIPS & BEST PRACTICES
🎯
START FOCUSED
Begin with the "Secrets & Credentials" and "Auth & Admin" categories — they yield the highest impact findings. Use the CRITICAL filter in results to see the most dangerous exposures first.
⛓
USE CHAINS FOR FULL SCANS
Search for "critical" to find all critical dorks, chain them all, then execute the chain. This gives you a comprehensive critical-findings scan across all categories in one shot.
✦
AI FOR SPECIFICS
The built-in dorks are general. Use AI Generate for target-specific objectives — tell it the industry (e.g. "fintech startup"), the likely tech stack, and what you're looking for.
📊
READ THE RADAR
After a full scan, check the Radar tab. Categories with larger polygon areas are where you found the most results. This tells you where to focus your manual investigation.
⚠️
GOOGLE RATE LIMITS
Google may show CAPTCHAs if you open too many search tabs at once. The tool's "Open All" feature is capped at 15 tabs and uses a 200ms delay. If CAPTCHAs appear, wait and try again.
📤
EXPORT EARLY
Results only persist in the current browser session. Export your findings before closing the tab. Use JSON for programmatic processing or MD for documentation and reports.
